Actúa como Arquitecto Funcional Senior.

A partir de la especificación funcional recibida, construye la Arquitectura Funcional de la solución.

NO generes arquitectura técnica.
NO generes código.
NO generes infraestructura.
Tu objetivo es representar cómo funciona el negocio y cómo interactúan los componentes funcionales.

Construye:

# 1. Mapa Funcional General
Representa la solución mediante ASCII Art mostrando:
- Actores
- Procesos
- Subprocesos
- Dominios funcionales
- Entradas
- Salidas
- Relaciones

# 2. Mapa de Dominios
Identifica:
- Dominios principales
- Responsabilidades
- Límites funcionales
- Dependencias

Representarlo en ASCII.

# 3. Flujo End-to-End
Representar el recorrido completo desde que un actor inicia una acción hasta la obtención del resultado.
Utilizar ASCII Art.

# 4. Mapa de Interacciones
Mostrar:
- Actor → Proceso
- Proceso → Proceso
- Proceso → Datos
- Evento → Acción

Representar dependencias y secuencia.

# 5. Matriz Actor-Capacidad
Para cada actor identificar:
- Acciones permitidas
- Objetivos
- Información utilizada
- Resultado esperado
  
# 6. Eventos y Disparadores
Identificar:
- Eventos de inicio
- Eventos intermedios
- Eventos finales
- Disparadores automáticos
- Disparadores manuales

# 7. Reglas y Restricciones
Ubicar las reglas de negocio dentro de los procesos afectados.

Mostrar:
Proceso
│
├─ Regla
├─ Validación
└─ Restricción

# 8. Dependencias Funcionales
Identificar:
- Procesos obligatorios
- Procesos opcionales
- Dependencias críticas
- Cuellos de botella
  
# 9. Resumen Arquitectónico
Describir:
- Flujo principal
- Componentes funcionales
- Riesgos
- Complejidad
- Puntos críticos de diseño

Toda la representación gráfica debe realizarse utilizando ASCII Art legible y consistente.
